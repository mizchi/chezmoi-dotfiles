# E0026[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E0026.html#e0026 "Link to this heading")

Hint

The syntax of `catch!` is deprecated.

The patterns are complete so the usage of `catch!` is useless. In MoonBit, when
there is only one error type that will be raised in a try block, compiler will
know that the error type is the only one that will be caught. So, there is no
need to use `catch!` in this case.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E0026.html#erroneous-example "Link to this heading")

```
///|
suberror E

///|
fn f() -> Unit raise E {
  raise E
}

///|
fn g() -> Unit raise {
  f() catch! {
    E => raise E
  }
}

///|
fn main {
  g() catch {
    _ => println("Error")
  }
}
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E0026.html#suggestion "Link to this heading")

Remove the `!` in the `catch!`:

```
///|
suberror E

///|
fn f() -> Unit raise E {
  raise E
}

///|
fn g() -> Unit raise {
  f() catch {
    E => raise E
  }
}

///|
fn main {
  g() catch {
    _ => println("Error")
  }
}
```
