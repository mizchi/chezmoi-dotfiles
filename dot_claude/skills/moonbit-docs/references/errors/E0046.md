## Contents

# E0046[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E0046.html#e0046 "Link to this heading")

Useless `..` in struct or constructor pattern.

`..` can be used to ignore unmatched fields in struct pattern, or ignore
unmatched labelled arguments in constructor pattern. But if all fields/labelled
arguments are already matched, the `..` is useless, and this warning will be
emitted.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E0046.html#erroneous-example "Link to this heading")

```
///|
struct Point {
  x : Int
  y : Int
}

///|
fn f(p : Point) -> Unit {
  let { x, y, .. } = p
  println(x + y)
}

///|
test {
  f({ x: 1, y: 2 })
}
```
