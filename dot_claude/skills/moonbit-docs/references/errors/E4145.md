# E4145

## Contents

# E4145[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4145.html#e4145 "Link to this heading")

Cannot implement trait because it is sealed.

Sealed trait means the trait is not allowed to be implemented in other packages.
Use `pub(open)` to make the trait open to be implemented by other packages.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4145.html#erroneous-example "Link to this heading")

Suppose there is a package `a` in module `username/hello`:

`a/moon.pkg.json`:

```
{}
```

`a/a.mbt`:

```
pub trait Sealed {
  to_int() -> Int
}
```

While in package `b`:

`b/moon.pkg.json`:

```
{
  "import": [
    "username/hello/a"
  ]
}
```

`b/b.mbt`:

```
type A Int

impl @a.Sealed for A with to_int(self : A) -> Int { // Error: Cannot implement trait '@a.Sealed' because it is readonly.
  self._
}
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4145.html#suggestion "Link to this heading")

You can change the visibility of the trait to `pub(open)`, so that it can be
implemented by other packages.

```
pub(open) trait Sealed {
  to_int() -> Int
}
```
