# E4156[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#e4156 "Link to this heading")

extern "A" is unsupported in B backend.

This error happens when an FFI function written for a specific backend is
discovered during the compilation of a different backend.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#erroneous-example "Link to this heading")

```
///|
pub extern "C" fn getchar() -> Int = "getchar"
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#suggestion "Link to this heading")

You can use conditional compilation to control when the function is available.

* If you want a file level conditional compilation, configure the
  [targets](https://docs.moonbitlang.com/en/stable/toolchain/moon/package.html#conditional-compilation) field.

  moon.pkg.json[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#id1 "Link to this code")

  ```
  {
    "targets": {
      "top.mbt": ["native"]
    }
  }
  ```

  top.mbt[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#id2 "Link to this code")

  ```
  ///|
  pub extern "C" fn getchar() -> Int = "getchar"
  ```
* If you want a function level conditional compilation, add the cfg attribute.

  other.mbt[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4156.html#id3 "Link to this code")

  ```
  ///|
  #cfg(target="native")
  pub extern "C" fn getchar2() -> Int = "getchar"
  ```
