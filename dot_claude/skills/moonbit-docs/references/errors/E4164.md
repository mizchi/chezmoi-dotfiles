# E4164[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4164.html#e4164 "Link to this heading")

Type declaration mismatch with mbti.

When a package implements a
[virtual package](https://docs.moonbitlang.com/en/stable/toolchain/moon/package.html#virtual-package), it is checked
against the interface of that virtual package, and this error is emitted if some
of the type declarations have different arities.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4164.html#erroneous-example "Link to this heading")

E4160.mbti[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4164.html#id1 "Link to this code")

```
package "moonbit-community/E4164";

type Foo[_]
```

top.mbt[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4164.html#id2 "Link to this code")

```
///|
pub enum Foo {}
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4164.html#suggestion "Link to this heading")

Modify the implementation such that it has the compatible signature as the
virtual package it declared to implement.

```
///|
pub enum Foo[_] {}
```
