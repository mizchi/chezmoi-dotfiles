## Contents

# E4000[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4000.html#e4000 "Link to this heading")

Generic type variable name is already used.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4000.html#erroneous-example "Link to this heading")

```
struct Container[T, T] {
  value : T
}

fn transform[A, A](https://docs.moonbitlang.com/en/stable/language/error_codes/x : A) -> A {
  x
}
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4000.html#suggestion "Link to this heading")

Use different names for type variables:

```
struct Container[T1, T2] {
  value : T1
}

fn transform[A, B](https://docs.moonbitlang.com/en/stable/language/error_codes/x : A) -> B {
  // ... implementation
}
```

Or remove the duplicate type parameter if you meant to use the same type:

```
struct Container[T] {
  value : T
}

fn transform[A](https://docs.moonbitlang.com/en/stable/language/error_codes/x : A) -> A {
  x
}
```
