# E4160[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4160.html#e4160 "Link to this heading")

Type mismatch with mbti.

When a package implements a
[virtual package](https://docs.moonbitlang.com/en/stable/toolchain/moon/package.html#virtual-package), it is checked
against the interface of that virtual package, and this error is emitted if some
of the interface has different types.

## Erroneous example[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4160.html#erroneous-example "Link to this heading")

E4160.mbti[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4160.html#id1 "Link to this code")

```
package "moonbit-community/E4160";

fn f() -> Unit
```

top.mbt[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4160.html#id2 "Link to this code")

```
///|
pub fn f() -> Int {
  42
}
```

## Suggestion[#](https://docs.moonbitlang.com/en/stable/language/error_codes/E4160.html#suggestion "Link to this heading")

Modify the implementation such that it has the exact signature as the virtual
package it declared to implement.

```
///|
pub fn f() -> Unit {

}
```
